# Data Model: App Documentation (Jekyll Site + In-App AI)

## Overview

This feature introduces no SwiftData persistent models. All data is either static bundle resources (HTML files, JSON index, PNG assets) or transient in-memory state. The entities below describe the logical data structures used at runtime and build time.

---

## Entity: DocPage (in-memory / bundle resource)

Represents a single documentation topic loaded from the app bundle.

| Field | Type | Notes |
|-------|------|-------|
| `id` | `String` | Filename stem, e.g. `"messages"` |
| `title` | `String` | Display title, e.g. `"Messages & Channels"` |
| `section` | `DocSection` | `.user` or `.developer` |
| `htmlURL` | `URL` | `Bundle.main.url(forResource: id, withExtension: "html", subdirectory: "docs/\(section.rawValue)")` |
| `keywords` | `[String]` | Populated from `index.json` at app launch |
| `charCount` | `Int` | Pre-computed character count for token budget estimation |

**Validation rules**: `id` must be non-empty and correspond to an existing bundle file. `htmlURL` must resolve — bundle integrity is verified in `DocBundleTests`.

**State transitions**: None — `DocPage` is immutable after loading from index.

---

## Entity: DocSection (enum)

```swift
enum DocSection: String, CaseIterable {
    case user       = "user"
    case developer  = "developer"
}
```

| Case | Bundle path | Site path |
|------|-------------|-----------|
| `.user` | `docs/user/` | `/user/` |
| `.developer` | `docs/developer/` | `/developer/` |

## Entity: DocBundle (singleton service)

Manages the full set of `DocPage` values loaded from the app bundle. Loaded once at app launch.

| Field | Type | Notes |
|-------|------|-------|
| `pages` | `[DocPage]` | All pages, ordered by section then title |
| `indexURL` | `URL` | `Bundle.main.url(forResource: "index", withExtension: "json", subdirectory: "docs")` |
| `bundleSizeBytes` | `Int` | Total bytes of all resources in `docs/` subdirectory |

**Loading**: `DocBundle.shared` reads `docs/index.json`, decodes the array, and constructs `DocPage` values. Falls back gracefully to empty if index is missing (SC-003 still met — TOC shows "No docs available").

---

## Entity: KeywordIndexEntry (Codable, build-time + runtime)

Codable struct matching the JSON schema in `contracts/keyword-index-schema.json`.

```swift
struct KeywordIndexEntry: Codable {
    let id: String
    let title: String
    let section: String          // "user" or "developer"
    let keywords: [String]
    let charCount: Int
}
```

**Generated by**: `scripts/build-docs.sh` → `Meshtastic/Resources/docs/index.json`  
**Consumed by**: `DocBundle.shared` at app launch; `AIDocAssistant` at query time.

## Entity: AIDocAssistant (in-memory, iOS 26+ only)

Transient service. No persistent state.

| Field | Type | Notes |
|-------|------|-------|
| `question` | `String` | User's free-text question |
| `retrievedPages` | `[DocPage]` | Top 2–3 pages matched by keyword scoring |
| `tokenBudgetUsed` | `Int` | Measured via `SystemLanguageModel.tokenCount(for:)` before querying |
| `response` | `String?` | Foundation Models response text |
| `isLoading` | `Bool` | True while awaiting model response |
| `error` | `Error?` | `LanguageModelSession.GenerationError` if context window exceeded or model unavailable |

**Retrieval algorithm**:
1. Tokenize `question` → lowercase terms.
2. For each `DocPage`, count matching `keywords`.
3. Sort descending by match count; take top 3.
4. Measure combined `charCount / 3.5` ≈ token estimate. If over 3,000, drop the lowest-scoring page.
5. Pass page text as context to `LanguageModelSession`.

**Error handling**: On `.exceededContextWindowSize`, reduce context to top 1 page and retry once. On model unavailability, show fallback message and offer to open the doc page directly.

## Entity: DocVersionManifest (CI / Jekyll, build-time)

Represents `_data/versions.yml` entries updated by the release workflow.

```yaml
- version: "2.8.0"
  path: "/v2.8.0/"
  label: "2.8.0 (Latest)"
  is_latest: true
- version: "beta"
  path: "/beta/"
  label: "Beta (Pre-release)"
  is_prerelease: true
```

**Updated by**: `docs-release.yml` (on `v*.*.*` tag) and `docs-deploy.yml` (on push to `main`).

## Build-Time Artefacts (not app entities)

| Artefact | Location | Generated by | Consumed by |
|----------|----------|-------------|------------|
| GFM source files | `docs/user/*.md`, `docs/developer/*.md` | Hand-authored | `build-docs.sh` |
| Generated HTML | `Meshtastic/Resources/docs/**/*.html` | `build-docs.sh` | Xcode Copy Files → app bundle |
| CSS stylesheet | `Meshtastic/Resources/docs/assets/docs.css` | Hand-authored | Generated HTML via `<link>` |
| Keyword index | `Meshtastic/Resources/docs/index.json` | `build-docs.sh` | `DocBundle.shared` |
| Screenshot PNGs | `Meshtastic/Resources/docs/assets/screenshots/*.png` | `copy-snapshots.sh` | Doc HTML `<img>` tags |
| Jekyll site output | `_site/` (gitignored) | GitHub Actions | `actions/deploy-pages` |
| `_data/versions.yml` | `docs/_data/versions.yml` | Release workflow | Jekyll version selector |
