# Doc Translations

Machine-translated documentation files generated by the Meshtastic Apple app using Apple's on-device Translation framework.

## Structure

```
translations/
  {language_code}/
    {app_version}/
      user/
        getting-started.md
        bluetooth.md
        ...
      developer/
        architecture.md
        ...
```

## How it works

1. The app translates documentation pages on-device when the user's language differs from English
2. Translated markdown files are cached locally on the device
3. The `push-translations.yml` GitHub Action (manual trigger) pushes translations to the `meshtastic/meshtastic` docs site repo as a PR

## Running the GitHub Action

1. Go to **Actions → Push Doc Translations to Docs Site**
2. Click **Run workflow**
3. Enter the language code (e.g., `fr`, `es`, `de`) and app version (e.g., `2.7.13`)
4. The action checks if translations already exist on the docs site or have an open PR
5. If not, it creates a branch and opens a PR with the translated files

## Adding translations

To add translated files here:

1. Run the app with a non-English device language
2. Browse through the documentation pages (this triggers translation + caching)
3. Export cached translations from the app's Application Support directory
4. Place them in the appropriate `translations/{lang}/{version}/{section}/` folder
5. Commit and push to this repo
6. Run the GitHub Action to create a PR on the docs site

## Review

All machine translations should be reviewed by native speakers before being merged into the docs site.
